# T-Dash 手动验证清单

本文档用于本地工具链安装后的人手验证。当前项目以 Flutter 手机仪表盘 App 为主，PWA 原型作为参考验证面。

## 1. 当前本地工具状态

已准备到 `.toolchains/` 的本地工具：

- Flutter SDK：`.toolchains/flutter`
- JDK 21：`.toolchains/jdk`
- Android command-line tools：`.toolchains/android-sdk/cmdline-tools/latest`
- Chrome for Testing：`.toolchains/chrome/chrome-linux64/chrome`

`tools/check-m0.sh` 已配置为优先使用这些本地工具，并自动设置：

- `JAVA_HOME`
- `ANDROID_HOME`
- `ANDROID_SDK_ROOT`
- `CHROME_EXECUTABLE`
- Flutter / Android / Java / Chrome 的本地 PATH

## 2. 仍需人工确认的工具步骤

### 2.1 接受 Android SDK License 并安装 SDK 包

Android SDK 的 license 需要由开发者本人确认。进入仓库根目录后执行：

```bash
cd "/mnt/c/Users/ASUS/Documents/NewProject"
```

先接受 license：

```bash
env -u HTTP_PROXY -u HTTPS_PROXY -u NO_PROXY \
  HOME=/tmp/tdash-home \
  JAVA_HOME="$PWD/.toolchains/jdk" \
  JAVA_TOOL_OPTIONS=-Duser.home=/tmp/tdash-home \
  PATH="$PWD/.toolchains/jdk/bin:$PATH" \
  "$PWD/.toolchains/android-sdk/cmdline-tools/latest/bin/sdkmanager" \
  --sdk_root="$PWD/.toolchains/android-sdk" \
  --licenses
```

看到 license 提示后，阅读并输入 `y` 接受。

然后安装 Flutter 3.41 当前需要的 Android SDK 包：

```bash
env -u HTTP_PROXY -u HTTPS_PROXY -u NO_PROXY \
  HOME=/tmp/tdash-home \
  JAVA_HOME="$PWD/.toolchains/jdk" \
  JAVA_TOOL_OPTIONS=-Duser.home=/tmp/tdash-home \
  PATH="$PWD/.toolchains/jdk/bin:$PATH" \
  "$PWD/.toolchains/android-sdk/cmdline-tools/latest/bin/sdkmanager" \
  --sdk_root="$PWD/.toolchains/android-sdk" \
  "platform-tools" \
  "platforms;android-36" \
  "build-tools;36.0.0" \
  "ndk;28.2.13676358"
```

### 2.2 可选：安装 Linux 桌面工具链

当前 App 主要目标是手机端。Linux 桌面构建不是 M1/M2 必需项。

如果要让 `flutter doctor` 的 Linux desktop 项也通过，需要在 WSL 里输入 sudo 密码安装系统包：

```bash
sudo apt-get update
sudo apt-get install -y clang ninja-build pkg-config libgtk-3-dev
```

当前环境中 `sudo -n apt-get update` 返回 `sudo: a password is required`，所以这一步需要人工输入密码。
