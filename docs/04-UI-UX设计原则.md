## 1. 设计哲学

### 1.1 三条核心原则

**1. 仪表，不是 App**

主仪表盘要让用户在车内 1 秒内识别关键信息。这意味着：

- 大字号、高对比、信息密度低
- 图形化优于文字（车门状态用门图标而非"前左门已关"）
- 动效服务于状态变化，不为视觉炫技

**2. 暗色为主，致敬但不抄袭**

特斯拉车机以暗色 + 极简白红配色为标志性视觉。T-Dash 主基调采用暗色，但避免直接照搬车机配色与字体，建立可识别但不混淆的独立视觉。

**3. 诚实**

凡有数据来源差异（如 GPS 测速 vs 车机车速、BLE 信号弱），UI 必须诚实标注。不靠模糊处理伪造完整体验。

### 1.2 反例：避免做这些

- ❌ 复刻特斯拉车机界面（侵权 + 误导）
- ❌ 弹窗推送、营销 Banner、积分签到（与定位冲突）
- ❌ 复杂多级菜单（车内场景不友好）
- ❌ 浅色主题作为默认（眩光问题）
- ❌ 过度动画（影响仪表数据可读性）

## 2. 设计系统

### 2.1 颜色系统

#### 暗色主题（默认）

| 角色 | Token | 值 | 用途 |
| --- | --- | --- | --- |
| 背景主 | `bg-primary` | `#0A0E14` | 全屏背景 |
| 背景次 | `bg-secondary` | `#141A22` | 卡片底色 |
| 背景层级 3 | `bg-tertiary` | `#1E2530` | 嵌套卡片、按钮底 |
| 文字主 | `text-primary` | `#F5F7FA` | 主要数字、标题 |
| 文字次 | `text-secondary` | `#9BA3AE` | 单位、辅助说明 |
| 文字弱 | `text-tertiary` | `#5C6370` | 占位、disabled |
| 强调主 | `accent-primary` | `#3DDC97` | 电量、健康状态、CTA |
| 强调次 | `accent-secondary` | `#5DA0FF` | 信息提示、链接 |
| 警告 | `warning` | `#FFB547` | 胎压低、电量低 |
| 危险 | `danger` | `#FF5C5C` | 错误、重要警告（参考但不照搬特斯拉红） |
| 充电 | `charging` | `#3DDC97` 闪烁 | 充电中专用 |
| 描边 | `border` | `#2A3340` | 卡片描边、分割线 |

#### 浅色主题（可选）

仅在用户手动开启时启用，颜色对应反转。**不在车内场景推荐**。

### 2.2 字体系统

| 角色 | 字号 / 字重 | 字体 |
| --- | --- | --- |
| 仪表盘大数字（车速） | 96sp / Bold | SF Pro Display / HarmonyOS Sans Bold |
| 仪表盘中数字（电量、续航） | 48sp / Bold | 同上 |
| 标题 | 20sp / SemiBold | 系统默认 |
| 正文 | 14-16sp / Regular | 系统默认 |
| 标签 / 单位 | 11-13sp / Medium | 系统默认 |

**关键**：避免使用特斯拉车机的 Gotham 字体；优先使用系统默认字体（SF Pro / HarmonyOS Sans / Roboto）保持平台原生感。

### 2.3 间距系统

8 倍数栅格：4 / 8 / 12 / 16 / 24 / 32 / 48 / 64

### 2.4 圆角

| 角色 | 半径 |
| --- | --- |
| 卡片 | 16dp |
| 按钮 | 12dp |
| 输入框 | 8dp |
| 图标背景 | 8dp |

### 2.5 阴影

暗色主题不使用传统阴影（不显著），用**1dp 描边 + 微弱 inner glow** 代替层级感。

### 2.6 图标

- 风格：线性图标，2dp 描边
- 尺寸：24dp（导航/操作）/ 32dp（状态卡片）/ 48dp（主仪表盘标记）
- 来源：自绘 + Lucide / Phosphor 图标库（开源，商用免费）
- **禁用**：直接复用特斯拉官方图标资源

## 3. 关键页面线框与交互

### 3.1 启动 → 首次配对流程

```
┌─────────────┐     ┌─────────────────┐     ┌────────────────┐
│   启动页     │ ──▶ │ 隐私政策同意页   │ ──▶ │ 添加车辆引导   │
│  (LOGO+slogan)│     │  (必须勾选同意) │     │ "靠近车辆并解锁"│
└─────────────┘     └─────────────────┘     └────────────────┘
                                                     │
                            ┌────────────────────────▼─┐
                            │  扫描车辆 BLE              │
                            │  (动画 + 找到设备列表)      │
                            └────────────────┬─────────┘
                                             │
                            ┌────────────────▼─────────┐
                            │  车机确认提示              │
                            │ "请在车机屏幕上确认配对"   │
                            │  (插图说明)                │
                            └────────────────┬─────────┘
                                             │
                            ┌────────────────▼─────────┐
                            │  配对成功 → 进入主仪表盘   │
                            └──────────────────────────┘
```

**关键交互点**：

- 隐私政策同意：必须主动勾选才能继续，符合 PIPL
- 配对引导插图：手绘卡片靠近中控台示意（不照搬车机界面）
- 失败处理：清晰文案 + 重试按钮 + 帮助链接

### 3.2 主仪表盘（核心页面）

```
┌──────────────────────────────────────────────────┐
│ ☰    T-Dash                          🔋 86%  ●连接 │  ← 顶部状态条
├──────────────────────────────────────────────────┤
│                                                  │
│                                                  │
│              ╱─────────╲                          │
│            ╱     58      ╲                       │  ← 主车速大数字
│           │    km/h GPS   │                       │     (含数据源标注)
│            ╲             ╱                       │
│              ╲─────────╱                          │
│                                                  │
│         ▓▓▓▓▓▓▓▓▓▓░░░░░  86%                    │  ← 电量条
│         续航 412 km                              │
│                                                  │
├──────────────────────────────────────────────────┤
│  🌡 22°C       🔒 已锁       🚗 全部关闭          │  ← 状态卡片行
│  空调待机     车辆          门窗后备箱             │
├──────────────────────────────────────────────────┤
│  [ 解锁 ]  [ 空调 ]  [ 闪灯 ]  [ 更多 ]            │  ← 快捷控制
└──────────────────────────────────────────────────┘
```

**布局原则**：

- 车速占据视觉中心，是仪表盘的"主角"
- 电量第二优先级（横向条带 + 数字）
- 状态卡片图形化、单行展示
- 快捷控制贴底部（拇指可达）

**车速数据源标注**：

- v1 显示 "GPS"
- v2（接 OBD）显示 "CAN" 或不显示（与车机一致）
- 弱信号时显示 "信号弱"占位（不显示乱跳数字）

**交互原则**：

- 危险或不可逆操作（鸣笛、解锁、哨兵切换）需"长按 1.5 秒"或"二次确认"
- 命令发出后立即给出 Loading 状态，500ms 内有反馈
- 命令失败必须明确文案（"车辆超出蓝牙范围"vs"车辆未响应"）

### 3.4 状态详情页

主仪表盘点击任一状态卡片进入对应详情：

- 充电详情：充电中 / 完成时间 / 当前功率 / 电量曲线（v2 完整曲线）
- 胎压详情：四轮胎压数字 + 单位（kPa / psi 切换）
- 空调详情：温度调节 + 模式（座椅加热 v1 不实现，v1.5 评估）

### 3.5 重新配对流程（私钥丢失场景）

**触发场景**（详见《02-架构》§3.5）：用户卸载重装、清除数据、换手机、Keychain 不可恢复。

**UI 流程**

```
┌──────────────────────────────────────────┐
│  启动检测：发现车辆记录但凭证已失效         │
├──────────────────────────────────────────┤
│                                          │
│  🔐 配对凭证已失效                        │
│                                          │
│  为了您的车辆安全，原配对凭证不再有效，    │
│  您需要重新与车辆配对。                    │
│                                          │
│  原因可能是：                             │
│  · 您卸载了 App 或清除了数据              │
│  · 您更换了手机                           │
│  · 系统安全机制刷新了凭证                  │
│                                          │
│           [重新配对]    [稍后]            │
└──────────────────────────────────────────┘

┌──────────────────────────────────────────┐
│  步骤 1/3：删除旧配对                     │
├──────────────────────────────────────────┤
│  [图示：车机屏幕路径]                     │
│                                          │
│  请先在车机上删除旧的 T-Dash 配对：        │
│                                          │
│  1. 点击车机屏幕底部图标                   │
│  2. 进入"安全 → 钥匙管理"                  │
│  3. 找到名为 "T-Dash" 的卡片              │
│  4. 点击"删除"                            │
│                                          │
│  ▢ 我已完成上述步骤                       │
│                                          │
│           [上一步]    [下一步]            │
└──────────────────────────────────────────┘

→ 步骤 2/3：扫描车辆（同首次配对）
→ 步骤 3/3：车机确认（同首次配对）
→ 完成
```

**关键设计**

- 自动检测："凭证已失效"由 App 启动时 BLE 握手失败触发，无需用户判断
- 用户车辆昵称、单位偏好等不重置，仅刷新 vehicle_key_id
- 提示用户每次重新配对前必须删除旧 key（否则车机最多累积 100 张同名卡片）
- 完成后 Toast 提示"已重新连接您的车辆"

**反例：避免做这些**

- ❌ 直接弹出"配对失败"+ 重新进入首次配对流程（丢失上下文）
- ❌ 自动尝试用旧 vehicle_key_id 重连（必失败，浪费用户时间）
- ❌ 不告知用户车机上有累积旧 key（埋雷）

## 4. 交互原则

### 4.1 车内场景特殊考量

T-Dash 主要使用场景是用户**在车内**（驾驶位 / 副驾），手机放在支架或手持。这意味着：

| 原则 | 落地 |
| --- | --- |
| 单手可操作 | 主要操作在屏幕下半 60% 区域 |
| 防误触 | 重要操作需长按或二次确认 |
| 大点击区 | 最小 48dp 触摸目标 |
| 强眩光对比 | 暗色主题 + 高对比文字 |
| 防分心 | 不弹无关推送、不强制阅读长文 |

### 4.2 状态变化反馈

| 事件 | 反馈方式 |
| --- | --- |
| BLE 连接成功 | 顶部状态条变色 + 1 秒淡入提示 |
| BLE 断开 | 顶部状态条变红 + Toast |
| 命令发出 | 按钮内 Loading 旋转 |
| 命令成功 | 触觉反馈（轻震）+ 状态卡片更新 |
| 命令失败 | 触觉反馈（错误模式）+ 红色 Toast 含错误说明 |
| 数据更新 | 数字平滑滚动（仪表盘风格），避免跳变 |

### 4.3 错误文案原则

错误信息必须**告诉用户怎么办**，不只是抛错误码。

| 反例 | 正例 |
| --- | --- |
| "蓝牙错误 -3" | "未找到车辆，请确认蓝牙开启并靠近车辆" |
| "超时" | "车辆未响应，请检查是否在 5 米内" |
| "签名失败" | "配对凭证已失效，请重新配对" |

### 4.4 加载与等待

- 配对：进度条 + 文字步骤（"扫描车辆"→"建立连接"→"等待车机确认"）
- 命令：按钮内 Loading，最长 5 秒，超时改"重试"
- 状态拉取：骨架屏（不用纯 Spinner）

### 4.5 驾驶模式（运动状态适配，P0 安全设计）

**为什么必须做**：主仪表盘默认展示控制按钮（解锁、空调、闪灯等）。若车辆在行驶中（驾驶员操作手机），点击控制按钮会引诱**分心驾驶**。这同时是产品安全责任和审核合规问题——苹果 App Store Guideline 1.1（Objectionable Content：危及人身安全）和 2.5.4（Software Requirements）对涉及车辆控制的 App 审核较严，必须主动消解此类风险。

**驾驶检测策略**：

| 输入 | 阈值 | 用途 |
| --- | --- | --- |
| GPS 速度（已融合） | > 5 km/h 持续 3 秒 | 进入驾驶模式 |
| 加速度计幅值 | 持续抖动 + 速度上升 | 辅助判定 |
| 切换迟滞 | < 1 km/h 持续 5 秒 | 退出驾驶模式 |

> 切换迟滞设计避免在路口停车 / 等红灯时频繁切换 UI。

**两态 UI 行为**

```
停车模式（默认）                  驾驶模式（运动中）
┌─────────────────────────┐       ┌─────────────────────────┐
│  顶部状态条             │       │  顶部状态条（精简）     │
├─────────────────────────┤       ├─────────────────────────┤
│  车速 58 GPS            │       │                         │
│                         │       │   ╔═══════════════╗     │
│  电量条 86%             │       │   ║   车速 58     ║     │
│  续航 412 km            │       │   ║    km/h GPS   ║     │
│                         │       │   ╚═══════════════╝     │
│  状态卡片 ×3            │       │                         │
│                         │       │   电量 86%  ▓▓▓░░       │
│  快捷控制按钮 ×4        │       │                         │
│                         │       │   控制功能行驶中不可用    │
└─────────────────────────┘       └─────────────────────────┘
```

**驾驶模式下的禁用项**

- 禁用：所有控制按钮（解锁、空调、闪灯、备箱、哨兵）
- 禁用：导航到设置 / 配对 / 详情页（保留"返回"按钮）
- 禁用：弹窗 / Toast 中的可点击操作（仅文字提示）
- 保留：顶部连接状态、车速、电量、紧急情况下的文字提示

**用户提示**

- 进入驾驶模式：底部短暂出现"行驶中已隐藏控制按钮"Toast，2 秒淡出
- 用户尝试触摸已禁用按钮：触觉反馈（错误模式）+ 提示"请停车后再操作"
- 设置中提供"驾驶模式开关"（默认开启）—— 但即便用户关闭，也保留"长按 1.5 秒"二次确认作为兜底

**审核应对**

- App Store 提审时附说明：本 App 已实现驾驶模式，行驶中禁用所有控制操作
- 演示视频中包含驾驶模式切换片段
- 用户协议中明确"行驶中应以车辆仪表为准，App 仅作辅助"

## 5. 国际化与本地化（v1 简化）

| 项 | 处理 |
| --- | --- |
| 语言 | 简体中文为主，预留 i18n 结构 |
| 单位 | 公里 / 英里、摄氏 / 华氏 在设置中切换 |
| 数字格式 | 中文区使用阿拉伯数字 |

## 6. 无障碍

| 项 | 落地 |
| --- | --- |
| 文字最小尺寸 | 11sp（标签）、14sp（正文） |
| 对比度 | 主要文字 ≥4.5:1，大字号 ≥3:1 |
| 触摸目标 | 最小 48dp |
| VoiceOver / TalkBack | 关键控件添加 semantic label |
| 不依赖颜色传递信息 | 警告同时用图标 + 文字 + 颜色 |
| 暗色 / 浅色双适配 | 设置可切换 |
