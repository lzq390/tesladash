## 1. 产品概述

### 1.1 一句话定位

**一款为中国特斯拉车主打造的近场仪表盘 App，让你在车内随时看见关键状态、单手完成常用控制。**

### 1.2 产品愿景

特斯拉官方 App 强调"车不在身边时的远程操作"，但车主在车内、车边的"近场"场景反而是被忽视的：上车前想瞄一眼电量、坐进车里想看第二屏的车速和能耗、停车时想看充电进度、夜间想快速空调预热。T-Dash 专注解决这些近场体验空白，软件版即可使用，未来可通过配套 OBD 蓝牙适配器升级为完整驾驶数据中心。

### 1.3 核心价值主张

- **零账号门槛**：v1 纯软件，下载即用，不依赖云账号、不需注册官方车主 OAuth；核心功能（仪表盘、控制）全部离线可用
- **数据隐私**：车辆数据与位置数据全部本设备处理，不上传任何云端；唯一的网络流量是匿名崩溃监控（用户可关闭）
- **可成长**：v2 可选购 OBD 适配器解锁完整车机级数据
- **诚实**：明确告知数据边界，GPS 测速 vs 车机车速差异透明展示

## 2. 目标用户

### 2.1 主要用户画像

**特斯拉中国车主，Model 3 / Model Y 为主，年龄 25-45，一二线城市，对车机体验有讲究。**

具体细分：

- **新手车主（购车 0-6 个月）**：希望更熟悉车辆状态，喜欢看仪表数据
- **资深车主（1 年以上）**：对官方 App 不满，寻求第三方增强体验
- **极客 / 改装爱好者**：v2 阶段的核心硬件用户群，愿意为深度数据付费

### 2.2 用户核心需求（按优先级）

1. **车内第二屏仪表盘**：开车时手机支架上看车速、能耗、电量，比扭头看屏方便
2. **上车前/下车后状态查看**：靠近车后快速看一眼电量续航锁状态
3. **常用控制快捷化**：空调预热、闪灯找车、解锁，比官方 App 步骤少
4. **充电场景陪伴**：停车充电时实时看进度、剩余时间

### 2.3 非目标用户

- **要求远程查车/远程控制的用户**：v1/v2 都做不到，需明确告知避免负面评价
- **多车队用户、企业用户**：v1 仅支持单车
- **非特斯拉车主**：协议和数据完全不通用

## 3. 市场与竞品

### 3.1 市场体量（量级估算）

中国特斯拉保有量截至 2026 年初约 200 万辆量级，年新增 50 万级。可触达用户基数充足，付费用户群体相对集中。

### 3.2 主要竞品

| 竞品 | 数据来源 | 模式 | 主要差距 |
| --- | --- | --- | --- |
| 特斯拉官方 App | Fleet API（特斯拉云） | 免费 | 远程强、近场弱、数据维度受限 |
| 小特 | Owner API（合规存疑） | 免费+订阅 | 已被特斯拉多次限制；用户对稳定性投诉多 |
| Tesla助手类（多家） | 多为 Owner API | 免费+订阅 | 同上 |
| Stats / TezLab（海外） | Fleet API | 订阅 | 中国区下载困难，无中文 |

### 3.3 差异化

T-Dash 的差异化不是"功能更多"而是"路径不同"：**官方 BLE + 本地处理 + 近场专注**，避开了所有走 Owner API 的竞品的合规与稳定性风险。这点对应用商店审核、对用户信任、对长期可持续都是优势。

## 4. 功能范围

### 4.1 v1 功能清单（纯软件 + Tesla 官方 BLE）

#### 4.1.1 配对与连接管理（P0）

- 首次启动引导：扫码/手动添加车辆 → 在车机上确认配对 → 注册公钥
- 自动重连：进入 BLE 范围自动连接
- 多设备共存：与官方 App 等并行使用不冲突
- 连接状态可视化：清晰展示"已连接 / 范围外 / 未配对"三态

#### 4.1.2 仪表盘主页（P0）

核心信息一屏可见：

- **车速大数字显示**（GPS 测速，标注数据源）
- **电量百分比 + 续航里程**
- **空调状态**（车内温度、设定温度、运行状态）
- **车门 / 车窗 / 后备箱锁状态**（图形化）
- **充电状态**（充电中显示功率、剩余时间、充电口状态）
- **胎压**（部分车型支持）

设计要求：暗色主题为主、夜间模式、单手可达、数字/图标清晰可在车内有阳光时看清。

#### 4.1.3 基础控制（P0）

- 解锁 / 上锁
- 空调启动 / 关闭 / 设温度
- 闪灯 / 鸣笛（找车）
- 前备箱 / 后备箱开启
- 充电口开 / 关
- 哨兵模式开 / 关

设计要求：高频操作（解锁、空调）需在主页可达；危险操作（鸣笛）需二次确认。

#### 4.1.4 GPS 车速显示（P0，技术核心）

- 主数据源：手机 GPS
- 平滑处理：卡尔曼滤波 + 加速度计/陀螺仪传感器融合，目标在常规驾驶场景下贴近车机表读数（±2 km/h 内）
- 透明标注：UI 上明确标识"GPS 测速"，避免误导
- 失效处理：进入隧道/地库 GPS 丢失时显示"信号弱"占位，不显示乱跳数字
- 数据源抽象：架构上预留 v2 切换 CAN Provider 的接口，UI 不感知差异

#### 4.1.5 设置与帮助（P1）

- 单位制（km/h vs mph，km vs mi）
- 主题（深色 / 浅色 / 自动）
- BLE 配对管理 / 解绑
- 隐私政策 / 用户协议
- 数据来源说明（教育用户 GPS 测速边界）
- 反馈与帮助

### 4.2 v1 明确不做

- 远程查车、远程控制（BLE 范围限制）
- 行车记录、行驶轨迹（BLE 不暴露 GPS 流，必须等 v2 OBD）
- 能耗历史与分析（同上）
- 多车支持（单车足够覆盖 95% 个人车主）
- 社区、论坛、UGC（与产品定位不符）
- CarPlay / Android Auto 集成（评估后置 v1.5）

### 4.3 v2 规划（OBD 蓝牙适配器，独立产品）

售卖配套 OBD-II 蓝牙适配器（建议定价 ¥299-399），用户安装后 App 自动检测并解锁：

- **真实车机车速**（从 CAN 总线读取，替换 GPS 数据源）
- **行车记录与轨迹**（按行程自动记录、地图回放）
- **能耗分析**（瞬时功率、kWh/100km、加速/制动能量回收统计）
- **驾驶习惯报告**
- **详细车辆诊断**（DTC 故障码读取）

v2 需独立立项评估硬件供应链、认证（3C/CE/FCC）、售后等，本 PRD 仅描述软件侧切换逻辑。

## 5. 关键产品决策与假设

### 5.1 近场使用边界（已锁定）

明确接受 BLE 近场限制，不向用户承诺远程功能。优势是合规干净、零云端依赖。劣势是与官方 App 形成功能差。处理方式：在产品文案中清晰定位为"近场伴侣"而非"官方 App 替代品"。

### 5.2 数据源抽象架构（已锁定）

v1/v2 共用一套 UI 和业务逻辑，车速、轨迹等数据通过 Provider 接口注入。这是最关键的技术原则，详见《02-技术架构与选型》。

### 5.3 一次性付费定价（**假设性决策**，需数据验证）

**假设**：定价 ¥18-28 的一次性付费模式可在中国特斯拉车主群体中达到可接受的下载-付费转化率。

**风险**：中国 App 市场对纯付费下载历史接受度低，转化可能远低于预期。

**应急切换路径**：

- v1 上线后 90 天监控关键指标
- 若付费转化率 < 1%（行业较差水平）或 LTV < ¥10，触发 v1.5 转 Freemium
  - 仪表盘核心免费
  - 控制类功能解锁付费（一次性 ¥18 或订阅 ¥6/月）
  - 多主题、CarPlay、详细数据等增值
- v2 OBD 适配器作为独立硬件 SKU 不受此影响

### 5.4 不接入特斯拉云端（v1 已锁定）

不集成 Fleet API、不使用 Owner API。所有车辆数据来自 BLE 本地通信，所有位置数据本设备处理。这避免了 API 审批、汽车数据合规、特斯拉 ToS 三重风险。

> **唯一例外**：崩溃监控 SDK（Sentry 或 Bugly，待 W6 决策）会上传匿名崩溃堆栈和设备型号。用户可在设置中关闭。这条不构成"车辆数据上云"。

### 5.5 版本号语义（统一定义）

为避免后续沟通歧义，本项目版本号遵循以下定义：

| 版本 | 含义 | 触发条件 |
| --- | --- | --- |
| v1.0 | 软件 + BLE 首发 | M6 上架完成 |
| v1.x（v1.1 / v1.2 …） | 修复与小功能迭代 | 双周或月度发布 |
| v1.5 | 重大产品调整 | 商业模式切换（Freemium）/ CarPlay / 鸿蒙原生 等 |
| v2.0 | 软硬一体 | OBD 蓝牙适配器上市 + CAN 数据源接入 + 行车记录功能 |
| v2.x | v2 软件迭代 | 同上 |

**关键原则**：v2.0 必须与硬件产品同步发布；纯软件无 OBD 时不能跳到 v2 编号。

## 6. 商业模式

### 6.1 v1 定价

- **iOS 中国区 / 港澳**：¥18 一次性
- **华为 / 小米**：同价

### 6.2 v2 硬件

- OBD 蓝牙适配器：¥299-399（含 App 内升级）
- 已购 v1 用户购买硬件后自动激活高级功能

### 6.3 收入结构假设（v1 上线 6 个月）

| 项 | 假设 | 备注 |
| --- | --- | --- |
| 下载量 | 5-10 万 | 通过自然搜索 + 车主社群口碑 |
| 付费转化率 | 1-3% | 假设性，需验证 |
| 付费用户数 | 500-3000 | |
| 平均售价 | ¥18 | 扣除渠道分成后到手约 ¥12-13 |
| 6 个月毛收入 | ¥6k-39k | |

> **结论**：纯软件收入预期不构成商业闭环，**v2 硬件销售是真正的变现路径**。v1 的核心目标是**积累用户、验证 PMF、为 v2 硬件预热**。

## 7. 成功指标

### 7.1 v1 北极星指标

**月活跃用户中"每周打开 ≥3 次"的占比** —— 反映"近场伴侣"产品定位是否成立。目标 ≥30%。

### 7.2 子指标

| 指标 | 目标 | 频次 |
| --- | --- | --- |
| 配对成功率 | ≥95% | 每周 |
| 首次配对到主页打开 ≤5 分钟 | ≥80% 用户达成 | 每周 |
| GPS 车速与车机表偏差 ≤2 km/h | ≥90% 行程 | 上线后专项 |
| Crash-Free 用户率 | ≥99.5% | 每周 |
| 应用商店评分 | ≥4.3（iOS）/≥4.0（华米） | 每月 |
| 付费转化率 | 1-3%（假设） | 每月 |
| 客服问题响应 | ≤24h | 每周 |

### 7.3 v2 触发条件（评估硬件立项的门槛）

- v1 月活 ≥1 万
- 用户调研中 ≥40% 表达"愿为行车数据付费"
- 单人开发节奏稳定（v1 上线后 3 个月内有规律小版本）

## 8. 风险与依赖

### 8.1 技术风险

| 风险 | 概率 | 影响 | 缓解 |
| --- | --- | --- | --- |
| Tesla 修改 BLE 协议导致兼容失效 | 中 | 高 | 关注官方 SDK 更新；保留快速热修复能力 |
| GPS 测速精度不达预期 | 中 | 中 | 卡尔曼+IMU 融合；提前 2 周专项调优；UI 诚实标注边界 |
| Flutter BLE 插件在新机型/新系统兼容问题 | 中 | 中 | 至少 5 款主流机型测试；预留原生通道兜底 |
| 车型差异（Model S/X 老款支持度） | 中 | 中 | v1 仅明确支持 2021+ Model 3/Y |

### 8.2 合规风险

| 风险 | 缓解 |
| --- | --- |
| Tesla 商标使用引发法律纠纷 | 命名/图标避开主词；详见《03-合规专项》 |
| 苹果中国区审核拒绝 | 提前准备软著、隐私评估、与官方 App 差异说明 |
| 个人信息保护法合规 | v1 不收集个人信息（无注册无云端），最大限度规避 |

### 8.3 商业风险

| 风险 | 缓解 |
| --- | --- |
| 一次性付费转化率过低 | v1.5 Freemium 应急路径已预留 |
| 特斯拉官方功能更新覆盖近场体验 | 持续监控官方 App，保持 1-2 项核心差异化 |
| 单人开发节奏失控 | 严格 6 个月里程碑；优先级砍到底 |

### 8.4 关键依赖

- Tesla `vehicle-command` SDK（GitHub 开源）持续维护
- Flutter 生态 BLE 插件（`flutter_reactive_ble` 或 `flutter_blue_plus`）
- Apple App Store / 华为 / 小米开发者账号
- 软件著作权登记（建议 v1 上线前完成）
